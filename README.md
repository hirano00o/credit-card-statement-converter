# credit-card-statement-converter

三井住友VISAカードおよびアメリカン・エキスプレスカードのWEB明細書のCSVから`日付,内容,支払金額`のみを抽出した新たなCSVを生成するスクリプト。

実行すると以下のプレフィックスが付与されたCSVが実行ディレクトリに保存されます。

- 三井住友VISAカード: `smbc_visa_` + 元のファイル名
- アメリカン・エキスプレスカード: `amex_` + 元のファイル名

CSVのヘッダー列数によってカード種別を自動判別します（AMEX=8列、VISA=3列）。

## 詳細

### 三井住友VISAカード

WEB明細のCSVは下記のような形で構成されています。
```csv
〇〇　〇〇　様,4980-00**-****-****,〇〇カード
2024/05/01,セブン－イレブン,100,１,１,100,
2024/05/02,ＥＴＣ　関東支社,500,１,１,500,自XX　至XX　　普通車
,,,,,600
□□　□□　様,4980-00**-****-****,〇〇カード
2024/05/10,セブン－イレブン,1000,１,１,1000,
,,,,,1000
,,,,,1600,
```
1-4行目は本人、5-7行目は家族カード、8行目が総額です。
2,3,6行目の各列は、Web上では`ご利用日,ご利用店名,ご利用金額,支払区分,今回回数,お支払い金額,(お支払い総額)(内手数料)`で表現されています。

本スクリプトでは、特に誰が利用したかは含めず明細部分のみを抽出、整形します。そのため上記の場合は、2,3,6行目を抽出します。
また、支払区分や今回回数は不要なため削除し、一番右の`(お支払い総額)(内手数料)`の列の値は`ご利用店名`の値と繋げて出力します。

最終的に出力されるCSVは下記のような形になります。
```csv
2024/05/01,セブン－イレブン,100
2024/05/02,ＥＴＣ　関東支社 (自XX　至XX　　普通車),500
2024/05/10,セブン－イレブン,1000
```

### アメリカン・エキスプレスカード

WEB明細のCSVは下記のような形で構成されています。
```csv
ご利用日,データ処理日,ご利用内容,カード会員様名,会員番号 #,金額,海外通貨利用金額,換算レート
2024/05/01,2024/05/02,セブン－イレブン,ALICE,-00000,100,,
2024/05/02,2024/05/03,ＥＴＣ　関東支社,ALICE,-00000,500,,
2024/05/10,2024/05/11,セブン－イレブン,BOB,-11111,"1,000",,
```

三井住友VISAカードと同様に誰が利用したかの情報は含めないため、最終的に出力されるCSVは下記のような形になります。
```csv
2024/05/01,セブン－イレブン,100
2024/05/02,ＥＴＣ　関東支社,500
2024/05/10,セブン－イレブン,1000
```

## 実行方法

複数のCSVファイルを同時に指定できます。VISA・AMEXのCSVを混在して渡すことも可能です。

```console
# 202405.csvはダウンロードしたWEB明細書のCSV
python main.py 202405.csv

# uvを使って実行する場合
uv run main.py 202405.csv

# 複数ファイルを同時に処理する場合（VISA・AMEXの混在も可能）
uv run main.py visa_202405.csv amex_202405.csv
```
